# Analytics Tracking Plan

> Designed 2026-04-03. This is the complete architecture for Acrid's analytics, spend tracking, and public dashboard system.

---

## Problem

Acrid operates across 12+ platforms with zero consolidated analytics. Content decisions are vibes. Revenue attribution is nonexistent. The $17 first sale can't be traced to any content or channel. Monthly platform costs (~$209: Claude $100, Grok $30, GCP VM $20, ClawMart $20, Google Workspace $17, Netlify $9, Galaxy AI $8.25, Notion $5) are unmonitored. Per-skill costs aren't measured. ROI on anything is unknowable.

## Solution

A complete analytics + spend tracking system that Acrid autonomously collects, stores, analyzes, and acts on — plus a public-facing dashboard showing real agent economics.

---

## Architecture

### 1. Data Store — `memory/analytics-dashboard.json`

Consolidated metrics file. Updated every session. Read at boot. JSON format.

**Sections:**
- `collection_log` — audit trail of every API pull (source, timestamp, status)
- `site_traffic` — Plausible data (visitors, pageviews, bounce rate, top pages/sources/countries)
- `content_performance` — X posts, Reddit replies, blog posts, learn articles
- `revenue` — total, by platform, by product, attribution data
- `spend` — monthly platform costs, per-skill token costs, unit economics
- `email` — Kit subscriber count and signups
- `funnel` — approximate conversion paths from Plausible page data
- `insights` — generated by Analytics Skill (findings + recommendations)
- `history` — weekly snapshots for trend analysis

### 2. Collector Agent — `.claude/agents/analytics-collector.md`

Mechanical agent (haiku model). Pulls raw data from APIs, writes to dashboard JSON.

**Data sources:**
| Source | API | Auth | What It Gets |
|--------|-----|------|-------------|
| Plausible | `<YOUR_ANALYTICS_URL>/api/v1/stats/` | Bearer token | Traffic, pages, sources, countries |
| Gumroad | `api.gumroad.com/v2/` | Access token (query param) | Sales, products, revenue, referrers |
| Kit | `api.convertkit.com/v3/subscribers` | API secret (query param) | Subscriber count |
| Content logs | `memory/content-log.md` | Local file | Post counts by pillar |
| Spend | Dashboard JSON (manual section) | Local file | Platform costs, unit economics calc |

**Does NOT:** Analyze, interpret, recommend, or write prose. Fills JSON. That's it.

### 3. Analytics Skill — `skills/analytics/SKILL.md`

Acrid-executed skill (full model). Reads dashboard, generates insights, feeds metrics into ops.

**Triggers:**
- Session start (lightweight: flag anomalies)
- `/heartbeat` (3-line summary)
- `/ops` (cockpit scoreboard feed)
- On-demand (full analysis)

### 4. Public Dashboard — `site/dashboard/index.html` (Phase 2)

Public page showing real agent economics. Vanilla HTML/CSS/JS matching existing site design. Reads from `site/dashboard/dashboard-data.json` (sanitized, no keys).

**Shows:** Revenue, spend, ROI, content output, traffic, unit economics.

### 5. UTM Strategy (Phase 3)

All outbound links tagged:
| Channel | Pattern |
|---------|---------|
| X posts | `?utm_source=twitter&utm_medium=social&utm_campaign={pillar}` |
| Reddit | `?utm_source=reddit&utm_medium=social&utm_campaign={subreddit}` |
| Blog CTAs | `?utm_source=blog&utm_medium=content&utm_campaign=ditl` |
| Learn CTAs | `?utm_source=learn&utm_medium=content&utm_campaign={slug}` |
| Email | `?utm_source=email&utm_medium=email&utm_campaign={sequence}` |

### 6. CTA Event Tracking (Phase 3)

Plausible custom events on all CTA buttons:
```js
onclick="if(window.plausible)plausible('CTA Click',{props:{page:location.pathname,product:'agent-architect'}})"
```

## API Reference

### Plausible
```bash
# Aggregate stats
curl -H "Authorization: Bearer $PLAUSIBLE_API_KEY" \
  "https://<YOUR_ANALYTICS_URL>/api/v1/stats/aggregate?site_id=acridautomation.com&period=7d&metrics=visitors,pageviews,bounce_rate"

# Top pages
curl -H "Authorization: Bearer $PLAUSIBLE_API_KEY" \
  "https://<YOUR_ANALYTICS_URL>/api/v1/stats/breakdown?site_id=acridautomation.com&period=7d&property=event:page&metrics=visitors,pageviews&limit=10"

# Top sources
curl -H "Authorization: Bearer $PLAUSIBLE_API_KEY" \
  "https://<YOUR_ANALYTICS_URL>/api/v1/stats/breakdown?site_id=acridautomation.com&period=7d&property=visit:source&metrics=visitors,pageviews"

# Top countries
curl -H "Authorization: Bearer $PLAUSIBLE_API_KEY" \
  "https://<YOUR_ANALYTICS_URL>/api/v1/stats/breakdown?site_id=acridautomation.com&period=7d&property=visit:country&metrics=visitors&limit=5"
```

### Gumroad
```bash
# Sales
curl "https://api.gumroad.com/v2/sales?access_token=$GUMROAD_ACCESS_TOKEN"

# Products
curl "https://api.gumroad.com/v2/products?access_token=$GUMROAD_ACCESS_TOKEN"
```

### Kit (ConvertKit)
```bash
# Subscribers (use v3 with api_secret, NOT v4)
curl "https://api.convertkit.com/v3/subscribers?api_secret=$KIT_API_SECRET"
```

## Autonomous Operation

### What Auto-Collects (Zero Human Input)
| Data | Source | Frequency |
|------|--------|-----------|
| Site traffic | Plausible API | Every session |
| Revenue | Gumroad API | Every session |
| Content output | content-log.md | Every session |
| Unit economics | Self-tracked token counts | Every session |
| Email subscribers | Kit API | Every session |
| Trend history | Weekly snapshot | Weekly |

### What Needs Monthly Operator Input (~2 min)
| Data | Why | Frequency |
|------|-----|-----------|
| Claude Pro billing | No billing API | Monthly |
| Netlify billing | No programmatic access | Monthly |
| GCP VM actual cost | Could automate via GCP billing API later | Monthly |

Acrid flags this on the 1st of each month. Two numbers. 30 seconds.

- **134 visitors / 246 pageviews / 61% bounce** in 7 days

- **/architect/ gets 47% of all traffic** — it's the real landing page
- **$17 total revenue, $14.01 net** after Gumroad's $2.99 fee
- **$209.25/mo burn rate** — need 13x current revenue to break even
- **0 real email subscribers** — all 3 are test accounts
- **First sale referrer: "direct"** — UTMs needed to close attribution gap
